# Persona Distiller V1.2 — 四步精炼法

把一个人的"灵魂"提炼成 Skill，让 AI 用这个人的风格思考和说话。

> **V1.1 → V1.2 升级点**：删除商业定价逻辑（开源免费），简化流程为四步，保留核心能力：素材质量门槛、增量分析、冲突检测、完整输出包。

---

## 核心工作流

```
素材输入 → 质量门槛 → 增量分析 → 冲突检测 → 完整输出包
```

---

## Step 1: 收集素材

向用户索要以下至少一类素材（越多越好）：

| 素材类型 | 示例 | 最低要求 | 质量观察点 |
|---------|------|---------|-----------|
| 文章/博客 | 公众号文章、知乎回答、博客帖子 | 3篇以上 | 段落结构、论证方式 |
| 聊天记录 | 微信聊天导出、社群发言 | 20条以上 | 回复节奏、表情使用 |
| 邮件/文档 | 工作邮件、内部文档 | 5封以上 | 称呼习惯、结尾格式 |
| 口述录音转写 | 语音转文字稿 | 1段以上 | 口头禅、语速感 |
| 自我描述 | 用户口述"我是什么风格" | 可选补充 | 主观认知 |

### 质量门槛（Quality Gate）

收到素材后，立即做快速评估：

**内容丰富度评分（0-100）：**
- 字数 < 200字 → 不合格，询问补充
- 字数 200-500字 → 60分，可接受但偏低
- 字数 500-2000字 → 75分，良好
- 字数 > 2000字 → 90分，优秀

**多样性加分：**
- 有 2 种以上素材类型 → +10分
- 有 3 种以上素材类型 → +15分

**低于 60 分的素材**：直接告知风险，不拒绝但要透明。
> "这段素材字数偏少（不到200字），用它蒸馏出来的 Persona 可能会在'语言节奏'和'知识边界'两个维度表现较弱。建议补充一些素材，或者我们用快问快答补足。"

---

## Step 2: 冷启动增量分析

**不要等所有素材收集完毕才开始分析。**

收到第一份素材，立即开始初步分析；每收到新材料，更新分析结论。最终结论是全量素材的加权融合。

### 量化节奏统计（每份素材必做）

对每份素材做以下统计：

```
1. 平均句长 = 总字数 / 句子数量
2. 句号频率 = 句号数量 / 总字数 × 100
3. 逗号频率 = 逗号数量 / 总字数 × 100
4. 段均句数 = 句子总数 / 段落总数
5. 感叹号频率 = 感叹号数量 / 总字数 × 100
6. 问号频率 = 问号数量 / 总字数 × 100
7. 省略号使用 = 有/无
```

统计结果以表格形式呈现，让用户确认感知是否准确。

**节奏风格判断规则：**
- 平均句长 < 15字 + 句号频率高 → **短句型**（类似"说话干脆"）
- 平均句长 > 30字 + 逗号多 → **长句型**（类似"写书面文章"）
- 感叹号 > 0.5% → **情感外放型**
- 问号 > 1% → **互动思辨型**
- 省略号 > 0.3% → **留白型/犹豫型**

### 六维特征提炼

在量化统计基础上，提炼6个维度的人文特征：

1. **语言节奏**（量化 + 感知）：短句/长句/长短交替？标点偏好？段落密度？
2. **口头禅/高频词**（Top 10）：出现频率最高的词/短语，标注标志性表达
3. **情感倾向**：冷静理性？热情外放？毒舌讽刺？温柔治愈？幽默自嘲？
4. **知识领域**（擅长 Top 3 + 谨慎领域）：什么话题说得深？说什么容易露怯？
5. **价值立场**（明确观点清单）：对这个人的核心立场建模，3-5条
6. **禁忌/雷区**（反向清单）：绝不说什么？什么会让ta说"这不是我"？

**分析输出格式（展示给用户确认）：**

```
## [人名] 风格分析报告 v1.0

### 量化节奏
- 语速感：短句型（均句长约12字）
- 标点风格：句号为主（92%），几乎不用感叹号
- 段落：短段落，每段2-4句

### 六维特征
1. 语言节奏：短句干脆，结论先行
2. 口头禅："说实话"、"核心是"、"这个逻辑..."
3. 情感倾向：冷静分析型，克制表达情感
4. 知识领域：产品策略 / 用户增长 / 数据分析
5. 价值立场：数据说话 > 直觉判断；用户价值 > 商业价值
6. 禁忌：不说"以用户为中心"这种空话；不说"我觉得"开头

[确认以上分析是否准确？如有偏差请指出，我据此调整后再生成。]
```

## Step 3: 冲突检测（Conflict Check）

生成 Skill 前，检查是否与已有 Persona 存在风格冲突：

1. **同名检测**：扫描 `~/.qclaw/skills/` 下是否存在同名 Skill
2. **相似度提示**：如果新 Persona 与某个已有 Persona 在"口头禅"或"情感倾向"维度高度重合（>70%），发出警告：
   > "注意：[人名A] 和 [人名B] 的风格相似度较高（口头禅重合度75%，情感倾向相同）。生成了两个 Skill 后，切换使用时可能会感到混淆。建议：只在 SOUL.md 中加一段'与XXX的区别'来区分。"

3. **用户决策**：
   - 继续生成 → 正常流程
   - 合并到已有版本 → 读取旧版，做增量更新
   - 放弃 → 停止流程

## Step 4: 生成完整 Persona Skill 包

输出一个完整的可安装 Skill 包：

```
[人名]-persona-v[版本号]/
├── SKILL.md          # 核心技能定义
├── SOUL.md           # 灵魂部分
├── README.md         # 安装说明 + 使用指南
├── demo.txt          # 3组示例对话
├── CHANGELOG.md      # 版本更新记录
└── install.bat       # Windows 一键安装脚本
```

### SKILL.md 关键规则

- `description` 格式：`"以[人名]的风格[做什么]。当用户说[触发词]时使用。"`
- SOUL.md 部分是核心——必须让 AI "成为"这个人，而不是"模仿"这个人
- 写入方式：用 qclaw-text-file 技能的脚本写入，确保编码正确（Windows: utf-8-sig）

### 输出路径规则

**文件夹命名格式：**
```
[人名]-persona-v[版本号]-[YYYYMMDD]
```
示例：`张伟-persona-v1.0-20260512`

**CHANGELOG.md 格式：**

```markdown
# [人名]-persona 更新记录

## v1.0 — 首次蒸馏
- 时间：[日期]
- 素材来源：[素材类型]
- 主要特征：[一句话总结]
- 已知局限：[生成时标注的短板]
```

### demo.txt 格式

```
# [人名] Persona — 示例对话

## 示例 1：被否定时
用户：我觉得这个方案挺好啊
[人名]：

## 示例 2：被夸奖时
用户：你这篇文章写得真棒
[人名]：

## 示例 3：遇到不懂的领域
用户：帮我写一段 Rust 代码
[人名]：
```

### install.bat 模板

完整的安装脚本模板见 `references/install.bat`。脚本功能：
- 自动从 SKILL.md 提取 Persona 名称
- 检查源文件是否存在
- 创建目标目录
- 复制所有文件
- 验证安装是否成功

## Step 5: 质量二次检查（生成后必做）

生成完毕后，对照检查清单逐项自审：

- [ ] **声音识别测试**：读一段输出，闭眼能听到这个人在说话吗？
- [ ] **口头禅验证**：包含 ≥2 个真实素材中出现过的标志性表达
- [ ] **情感一致性**：与素材情感倾向偏差不超过一个等级
- [ ] **知识边界**：明确标注擅长（≥3）+ 不擅长（≥2）
- [ ] **禁忌红线**：包含 ≥2 条"绝不做"声明
- [ ] **触发词安全**：description 中的触发词不会误触发已有 Skill
- [ ] **Demo 可信度**：demo.txt 中的说法，用户读了会说"是的，我会这样说"
- [ ] **版本号**：SKILL.md frontmatter 中有正确的 version 字段

## 快问快答模式（V1.1 优化版）

当用户没有素材时，用以下7问，比V1.0更精准：

1. **"你最常说的一句话是什么？"** （捕捉口头禅）
2. **"你说话喜欢短句还是长句？"** （定义节奏基础）
3. **"你最讨厌别人怎么跟你说话？"** （反向锚定禁忌）
4. **"给你三句话，你会怎么改写？"**
   → 同步给出三句测试句，让用户用自己的风格改写：
   - "这个项目做得不错" （正面评价）
   - "我不同意你的观点" （冲突处理）
   - "今天心情不好" （情感表达）
5. **"你在什么话题上最有底气？"** （知识锚点）
6. **"你觉得自己最独特的表达习惯是什么？"** （自我认知锚定）
7. **"如果有人模仿你说话，你觉得哪里最假？"** （精准找雷区）

7问拿到5问以上回答 → 进入 Step 3 生成。

## 迭代更新流程

当用户说"不像我"或"需要更新"时：

1. **收集反馈**：具体哪里不像？给一个具体场景和说法
2. **定位维度**：将反馈归类到六维中的某一维
3. **更新特征**：修改对应维度的新值
4. **版本递增**：v1.0 → v1.1 → v1.2 ...
5. **写 CHANGELOG**：记录本次修正内容和触发原因
6. **重新生成完整包**

## 高级技巧

**多源场景融合**：同一人有多个场景风格（工作 vs 生活），在 SOUL.md 中用 `## 场景：` 分段定义，生成时询问用户主要场景。

**风格稀释警告**：当素材中出现多种不一致风格时，标注"风格稀释风险"，告知用户混合风格可能导致 Persona 模糊。

**批量蒸馏**：每次蒸馏单独一个文件夹，不混放；文件夹命名 `[人名]-persona-v1.x`。
