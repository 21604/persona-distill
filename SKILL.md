# Persona Distiller — 三步蒸馏法

把一个人的"灵魂"提炼成 Skill，让 AI 用这个人的风格思考和说话。

## 核心工作流

```
素材输入 → 风格分析 → Skill 输出
```

### Step 1: 收集素材

向用户索要以下至少一类素材（越多越好）：

| 素材类型 | 示例 | 最低要求 |
|---------|------|---------|
| 文章/博客 | 公众号文章、知乎回答、博客帖子 | 3篇以上 |
| 聊天记录 | 微信聊天导出、社群发言 | 20条以上 |
| 邮件/文档 | 工作邮件、内部文档 | 5封以上 |
| 口述录音转写 | 语音转文字稿 | 1段以上 |
| 自我描述 | 用户口述"我是什么风格" | 可选补充 |

**如果用户没有现成素材**，启动「快问快答模式」：问5个问题快速捕捉风格特征（见下方）。

### Step 2: 风格分析

从素材中提炼6个维度的风格特征：

1. **语言节奏**：短句还是长句？爱用分号还是句号？段落长短？
2. **口头禅/高频词**：哪些词反复出现？有没有标志性表达？
3. **情感倾向**：冷静理性？热情外放？毒舌讽刺？温柔治愈？
4. **知识领域**：擅长什么？提到什么话题会来劲？
5. **价值立场**：对什么事有明确观点？什么会激怒ta？什么会让ta兴奋？
6. **禁忌/雷区**：绝不说什么？什么风格会让ta觉得"这不是我"？

**分析输出格式**：直接在对话中展示提炼结果，让用户确认或修正后再生成 Skill。

### Step 3: 生成 Persona Skill

使用 `references/persona-template.md` 中的模板生成 SKILL.md。

关键规则：
- SKILL.md 的 `description` 字段必须写明触发条件，格式：`"以[人名]的风格[做什么]。当用户说[触发词]时使用。"`
- SOUL 部分是核心——必须让 AI "成为"这个人，而不是"模仿"这个人
- 写入方式：用 qclaw-text-file 技能的脚本写入，确保编码正确
- 输出路径：让用户决定，默认 `~/Desktop/[人名]-persona/`

## 快问快答模式

当用户没有素材或想快速体验时，依次问这5个问题：

1. **你最常说的一句话/口头禅是什么？**
2. **如果用三个形容词描述你的说话风格，是哪三个？**
3. **你最讨厌的沟通方式是什么？**（反向定义风格）
4. **你在什么领域最有底气？**（定义知识锚点）
5. **给你一句普通的话，你会怎么改写？** → 提供："这个项目做得不错" 让用户用自己的风格改写

拿到回答后直接跳到 Step 3 生成。

## 质量检查清单

生成 Skill 前逐项确认：

- [ ] 语言节奏：读一段输出，闭上眼睛能"听到"这个人在说话吗？
- [ ] 口头禅：至少包含2个标志性表达
- [ ] 情感调性：和素材中的情感倾向一致
- [ ] 知识边界：清楚标注了"擅长"和"不擅长"
- [ ] 禁忌红线：写明了绝不做的事
- [ ] description 触发词：足够明确，不会误触发

## 高级技巧

**多源融合**：如果同一人有不同场景的风格（如：工作邮件 vs 朋友圈），在 SOUL.md 中分场景定义，用条件判断切换。

**持续迭代**：Persona Skill 装上后，用户说"不太像我" → 收集具体哪里不像 → 回到 Step 2 修正对应维度 → 重新生成。

**批量蒸馏**：用户要蒸馏多人 → 每人独立一个 Skill，避免风格串台。
